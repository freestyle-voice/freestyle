The Voice Agent plugin lets you talk to an AI assistant by voice. When your
dictation is meant for the agent, Freestyle runs an LLM turn **instead of typing
the text** and shows the conversation in a panel that expands out from the
floating pill.

> "agent, summarise the last thing I copied" → the agent replies in the pill panel
>
> _(dictating while a terminal is focused)_ "explain this error" → agent replies

It's also the reference implementation of the **pill panel** extension point:
any plugin can attach its own live UI to the pill the same way. See the
[pill panels section of the SDK reference](/sdk-reference#pill-panels) to build
your own.

## Install

```bash
npm install @freestyle-voice/plugin-agent
```

Or install from **Settings → Plugins** inside the app.

## How to use it

The plugin only intercepts dictation that looks like it's for the agent —
everything else is dictated into your focused app as normal.

1. Enable the plugin in **Settings → Plugins**.
2. Trigger a turn one of two ways:
   - Say **"agent"** at the start of your dictation (e.g. _"Agent, what's the
     capital of France?"_). The wake word is matched however speech-to-text
     punctuates it — "Agent,", "Agent.", or a bare "Agent …" all work, as do
     "hey agent" and "ok agent" — and it's stripped before the prompt is sent.
   - Or dictate while a **terminal or code editor** is focused (Terminal, iTerm,
     Warp, VS Code, …), where the whole utterance is the prompt.
3. The pill grows into a panel showing your message and the agent's reply. Keep
   talking to continue the thread.
4. Press the dictation hotkey again to dismiss the panel, or use **Close** in the
   panel header. **Clear** wipes the conversation.

Set a custom **System prompt** in the plugin's settings to change the agent's
persona or instructions.

## How it works

The plugin combines a server hook with a pill panel:

1. **Interception** — an `afterTranscribe` hook checks the transcript against the
   trigger rules. On a match it strips the `agent:` prefix and runs a turn with
   the LLM you've already configured, via the SDK's [`api.llm`](/sdk-reference#pluginllm)
   capability.
2. **Consume** — it calls `api.control.consume()`, so the rest of the pipeline is
   skipped and nothing is pasted into your focused app.
3. **Panel** — consuming a dictation signals the app to open the pill panel and
   forward the transcript to it as a `transcriptReady` event. The panel, built on
   `window.freestyle.pill`, re-fetches the conversation from the plugin's own API
   route and renders it live, tracking the pill's state as you speak.

<Note>If no LLM is available, the agent replies with a short "configure a model"
message instead. The plugin reuses your configured model and keys — there's no
separate provider to set up. **Signed-in Freestyle Cloud users get this too**, running
on Freestyle Cloud's managed LLM with no local model required.</Note>

## The pill panel

The conversation UI is a `contributes.pill` panel — a sandboxed page that the
host overlays on the pill window and sizes per the manifest's `expand`
dimensions. It subscribes to pill events through `window.freestyle.pill` so it
knows when you're recording and when a new turn is ready, and it can expand and
collapse itself. This is the same API any plugin can use to build its own pill
experience.

## Build from source

```bash
pnpm install
pnpm --filter @freestyle-voice/plugin-agent build
```

This runs `pkgroll --minify` (server-side plugin code) followed by `vite build`
(the pill panel UI).
